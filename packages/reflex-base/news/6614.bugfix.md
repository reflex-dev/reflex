Building a component package no longer rewrites `pyi_hashes.json`: the `reflex_base.utils.pyi_generator` build-hook entrypoint now only emits the `.pyi` stubs bundled in the wheel and leaves the hash registry untouched. The scanner also tolerates source-less modules (e.g. an empty `__init__.py`) instead of raising `OSError` on Python < 3.13.
