Component-returning `@rx.memo` again accepts the standard base `Component` props — `key`, `id`, `class_name`, `style`, `custom_attrs`, `ref` — without declaring an `rx.RestProp`, restoring compatibility with the legacy custom-component `@rx.memo` (notably setting `key` on memo components inside `rx.foreach`). A deprecation warning points at `rx.RestProp`, and this fallback will be removed in 1.0. Other undeclared props — including component identity fields like `tag` and `library` — still raise.
