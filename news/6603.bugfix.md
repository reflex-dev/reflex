Fixed `State.router.url.query` (and the rest of `router_data`) going stale after the URL was changed with `window.history.replaceState`/`pushState` (e.g. via `rx.call_script`). React Router's location does not observe direct history manipulation, so the query string and hash are now read from the live `window.location` when populating router data, while the pathname stays basename-relative so `frontend_path` is not applied twice. Embedded (mount target) apps continue to use the in-widget memory router.
